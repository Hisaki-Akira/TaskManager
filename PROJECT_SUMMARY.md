## 設計原則

### アーキテクチャ
- **コンポーネント指向**: 再利用可能な小さいコンポーネント
- **単一責任**: 各コンポーネントは1つの責任のみ
- **Props down, Events up**: データフローの明確化

### UI/UX
- **視覚優先**: 色とレイアウトで直感的に理解
- **最小限のクリック**: 必要な情報にすばやくアクセス
- **フィードバック**: ホバー、アニメーション、色の変化

### データモデル
- **シンプル**: 複雑な関連を避ける
- **拡張可能**: 将来の機能追加に対応
- **型安全**: TypeScriptで実行時エラーを削減

## デプロイ準備

### 必要条件
- HTTPサーバー（python -m http.server など）
- モダンブラウザ（Chrome, Firefox, Safari, Edge）
- インターネット接続（React CDN用）

### 起動方法

```bash
# リポジトリをクローン
git clone https://github.com/Hisaki-Akira/TaskManager.git
cd TaskManager

# HTTPサーバーを起動
python3 -m http.server 8080

# ブラウザで開く
open http://localhost:8080
```
